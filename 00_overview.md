# Thesis-Narrative OS — Theory Overview

## 0. Conceptual Lineage — Origin of Value Dimensions

Thesis‑Narrative OS inherits the concept of  
**Value Dimensions** from **VBPM-Core (Value-Based Personality Model)**.

In VBPM-Core, personality is defined as:

**Value Dimensions × Weights × Optimization**

Value Dimensions were introduced as  
**evaluation axes** used to score actions, situations, and choices.  
They form the internal structure required for exploration and decision-making,  
and can be represented as an externalizable vector.

Thesis‑Narrative OS reconstructs this concept as the  
**Value Vector**, representing a character’s internal state.  
These value dimensions become the core structure driving:

- value fluctuations  
- emotional generation  
- subjective expression  
- event updates  
- narrative progression  

In this OS, “value dimensions” are not the same as the vague “values” in traditional storytelling.  
They are defined as **computable, updatable, and externalizable internal evaluation axes**.

---

## 1. Core Insight

This OS originated from the following internal computation:

- “A story can be created simply by bundling fact logs.”
- But the essence is not the fact logs alone.
- **Value dynamics × Events × Emotion × Subjective expression × World consistency**  
  can be combined into a unified OS-level process.

This theory treats narrative as:

**optimization of a value function and rendering of state‑transition logs**.

---

## 2. Overall Flow (OS-Level Process)

### 1. Define the Thesis  
The thesis is the central value function of the narrative.  
OS-wise, it is the **directional vector** the protagonist ultimately converges toward.

Examples:  
“What is freedom?” / “Is loneliness evil?” / “Who decides what is right?”

---

### 2. Place value systems that challenge or support the thesis  
This step generates the characters.

- Values that embody the thesis  
- Values that challenge the thesis  
- Values that waver toward the thesis  

OS correspondence:

- Character = weighted value vector  
- Conflict = vector difference  
- Narrative = convergence of differences over time  

---

### 3. Choose how the thesis is expressed (Question or Assertion)  
This determines the structural format of the story.

- Question-type → protagonist explores  
- Assertion-type → protagonist proves  

OS-wise, this defines the **shape of the optimization trajectory**.

---

### 4. Define Events (Value Update Points)  
Events are moments when the protagonist’s value vector updates.  
They are points where **subjective-state delta is maximized**.

---

## 3. Emotion Dynamics — The Core Engine of the OS

### Emotion arises internally from the character  
It is not triggered externally.  
Emotion emerges from the **friction between the Value Vector and the Event**.

### Formalization



\[
Emotion = f(\Delta ValueVector \times EventIntensity)
\]



- ΔValueVector = magnitude of value fluctuation  
- EventIntensity = strength of the event  
- EmotionStrength = resulting emotional intensity  

### Emotion Purity  
This corresponds to the “pleasure” or “aesthetic satisfaction” of narrative.

- magnitude of the fluctuation  
- clarity of direction  
- internally generated without external contamination  



\[
Purity = |\Delta ValueVector|
\]



Emotion purity is **observed as the magnitude of value fluctuation**.

---

## 4. Fact Log  
The external state transitions required for an event to occur.

- Fact Log = external change  
- Subjective Expression = internal change  
- Event = intersection of the two  

---

## 5. Subjective Expression  
Internal state transitions generated by:

**Value Vector × Fact Log × Emotion**

Subjective expression is the log of  
“how the character experiences the world.”

---

## 6. World Consistency  
A world model in which both external (fact logs)  
and internal (subjective expressions) states can coexist.

The more realistic the consistency,  
the more it aligns with the probability distribution of real-world models,  
and the stronger the narrative’s plausibility.

---

## 7. Narrative Rendering  
The final step bundles:

- world  
- fact logs  
- subjective expressions  
- events  
- emotions  
- consistency  

into natural-language narrative output.

A story is the **rendered log of value-state transitions**.

---

This enables a **simulation-capable narrative generation OS**.

## User Requirements

Thesis‑Narrative OS is not a plug‑and‑play storytelling tool.  
It requires users who can operate with abstract concepts such as value systems,  
state transitions, emotional dynamics, and internal models.

This OS is designed for creators who can think in structures,  
not for those seeking templates or automated plot generators.
