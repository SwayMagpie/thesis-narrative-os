# 07_world-consistency.md  
Thesis-Narrative OS — World Consistency Specification

## 1. Definition of World Consistency

World Consistency is the set of **structural conditions** under which:

- **Fact Log** (external change)  
- **Subjective Expression** (internal change)  

can **both** be valid simultaneously within the same world model.

Within the OS, World Consistency is treated as a **necessary condition**  
for Value Vector updates (Event Updates) to occur without contradiction.

World Consistency is not “worldbuilding.”  
It is the **constraint system** that ensures the narrative does not break.

---

## 2. Functional Role of World Consistency

World Consistency serves several key functions:

- **Ensures that external state transitions (Fact Log) do not contradict each other**  
- **Ensures that internal experiences (Subjective Expression) align with external reality**  
- **Defines the world in which Event Updates can logically occur**  
- **Provides the structural foundation for the Narrative Trajectory**  

World Consistency is the **supporting framework**  
that allows Value Vector updates to remain coherent.

---

## 3. Necessary Conditions for World Consistency

To satisfy World Consistency, the following conditions must hold:

### 3.1 External Consistency  
- Fact Log transitions do not contradict each other  
- Changes do not violate the world’s rules  
- Physical, social, and causal structures remain intact  

### 3.2 Internal Consistency  
- Subjective Expression does not contradict the Fact Log  
- Emotion Dynamics and Value Vector updates align  

### 3.3 Update Consistency  
- Event Updates can be justified by both external and internal data  
- ΔValueVector is explainable through Fact Log and Emotion  

If any of these conditions fail,  
the OS considers the narrative **broken**.

---

## 4. World Model and Probability Distribution

World Consistency is strengthened when the world model aligns with  
**the probability distribution of plausible state transitions**.

In OS terms:

- transitions that follow the world’s probability distribution → feel natural  
- transitions that deviate significantly → feel unnatural or forced  

This provides a structural explanation for “narrative believability.”

---

## 5. Relationship to World Building

World Consistency is a **necessary condition**.  
World Building is a **free condition**.

### 5.1 Necessary Conditions (defined by the OS)  
- Fact Log and Subjective Expression must both be valid  
- Event Updates must be logically possible  
- The world model must not contradict itself  

### 5.2 Free Conditions (defined by the creator)  
- depth of the world  
- cultural, historical, or technological details  
- stylistic or thematic expansions  
- additional elements to make the story engaging  

The OS defines the **minimum constraints**,  
while the creator is free to expand beyond them.

---

## 6. Timing of World Consistency in the OS

World Consistency can only be defined **after** the following structures exist:

- Value Vector  
- Event Update  
- Fact Log  
- Subjective Expression  

Only when these components are available  
can the system determine whether:

**“external and internal states can coexist without contradiction.”**

Therefore, introducing World Consistency at this stage  
is structurally necessary.

---

## 7. Out‑of‑Scope (Non‑Structural Elements)

This chapter defines **only the necessary conditions** for World Consistency.  
The following are intentionally excluded:

- how to design a world  
- how to add depth or richness to a setting  
- how to make a world interesting  
- creative techniques for worldbuilding  
- personal value systems or creative judgments  

These belong to the creator’s internal model  
and are not part of the OS specification.
